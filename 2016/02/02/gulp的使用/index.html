gulp的使用

  • gulp
  • 代码压缩
  • 代码合并

什么是gulp？

gulp是一种自动化构建工具

gulp的优点

  • 易于使用

    -通过代码优于配置的策略，Gulp让简单的任务简单，复杂的任务可管理

  • 构建快速

    -利用Node.js流的威力，你可以快速构建项目并减少频繁的IO操作

  • 插件高质

    -Gulp严格的插件指南确保插件如你期望的那样简洁高质得工作。

  • 易于学习

    -通过最少的API,掌握Gulp毫不费力，构建工作尽在掌握：如同一系列流管道。

gulp的安装及js代码的合并与压缩

1.gulp的安装非常简单，可以先安装一个全局的gulp，终端命令：

sudo npm install -g gulp

2.建一个文件夹将gulp移动进来（也可直接安装在本文件下），终端命令：

npm install gulp

3.项目初始化，终端命令：

npm init

4.随意安装两个js文件(如：a.js 和 b.js),并在其中随意写一段js函数如：
a.js代码：

function(){
    console.log(‘我是a’)
}

和b.js代码：

fucntion(){
    console.log(‘我是b’)
}

5.安装合并js代码的module，终端代码：

nmp install gulp-concat --save-dev

6.安装压缩js代码的module，终端代码：

nmp install gulp-jsmin --save-dev

注释：(–save-dev 可以生成开发依赖，让依赖来源在初始化产生的文件package.js中显示出来)

7.配置gulpfile.js这个文件（文件名字必须为这个，不可变动）

8.在gulpfile.js文件中将gulp、concat、jsmin引进来,并给gulp任务

var gulp=require('gulp');
var concat=requrie('gulp-conat');
var min=require('gulp-jsmin');

gulp.task('default',function(){
    gulp.src('js/*js') //要合并的两个js文件的相对路径 *代表所有的js文件        .pipe(concat('new.js')) //合并js文件；new.js合成的js文件的文件名字，自行命名
    .pipe(min()) //压缩js文件
    .pipe(gulp.dest('newdir/')) // 生成的文件在在新的目录newdir下

})

9.在终端执行命令：

gulp或gulp default

如此连个js文件变合并完成，显示结果：

function(){console.log('我是a')}
function(){console.log('我是b')}

关于gulp.task的进一步说明

gulp.task('default',['a','b'],function(){})

gulp.task()有三个默认值即：default，数组（要执行的函数名字，以数组形式展现，回调函数）

在终端执行时可以单个函数执行如：

gulp a    //执行函数名为a的函数

或

gulp b   //执行函数名为b的函数

而当此时执行：

gulp

函数会先运行a函数和b函数，最后执行默认函数即（default）；
