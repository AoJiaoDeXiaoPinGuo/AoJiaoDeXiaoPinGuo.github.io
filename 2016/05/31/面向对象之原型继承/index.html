##面向对象的继承

继承的作用：

让一个对象的属性和方法可以另一个对象是用。

继承的目的：

简化代码

是用场景：

当在项目中需要对一个组件进行功能扩张，但扩展功能又不能影响组件原有的基本功能时。

比如：
小红在拥有与小明一样的属性和方法同时，还多了一个爱好的方法：逛街。

1.小明和小红都有共同有基本属性好方法。

2.小红多了一个爱好方法。

使用call或者apply实现继承

function Parent(name){
    this.name=name;
    this.type="人类";
}

Parent.prototype.say=function(){alert(this.name);};

function Child(){
    Parent.call(this,name);
    this.sex="boy";
}
var obj=new Child("小明");
alert(obj.name);//空
alert(obj.sex);//boy

这个方法无法继承原型对象上德属性和方法。

使用原型继承

function Parent(name){
    this.name=name;
    this.type="人类";
}

Parent.prototype.say=function(){
    alert(this.name);
}

function Child(name){
    this.name=name;
    this.type="学生";
}

Child.prototype=new Parent();
Child.prototype.constructor=Child;
var obj=new Child("小明");
// obj.say();
alert(obj.type);//学生
alert(obj.name);//小明

这个方法改变了原型对象上的属性。

原型继承的第二种方式

function Parent(name){
    this.name=name;
    this.type="人类";
}

Parent.prototype.say=function(){
    alert(this.name);
}

function Child(name){
    Parent.call(this,name);
    // this.name=name;    
}

function F(){}

F.Prototype=Parent.prototype;
Child.prototype=new F();
Child.prototype.constructor=Child;
var obj=new Child("小明");
// obj.say();
alert(obj.type);//人类
alert(obj.name);//小明

这个方法继承了原有的属性，还能在不改变原型对象属性的基础之上，实现自身方法属性的扩展。

###总结：
使用第二种原型继承的步骤：

1 先创建一个空函数
2 将原型的原型链赋予这个空函数的原型链
3 空函数的实例化对象为继承者的原型（即Child.prototype=new F()）
4 创建一个实力化对象。

###扩展（关于克隆的问题）

传值与传址

1.传值：简单数据类型之间的赋值为传值（jsvascript中的简单数据类型共有五种：Arrary String Boolean Null Undefined）

var a=1;
var b=a;

b=5;
alert(a);// 1

a将值直接复制给b，修改b的值并不会影响到a。
2.传址：复杂数据类型之间的赋值为传址。

var arr1=[1,2,3,4];
var arr2=arr1;

arr2[0]=10;
alert(arr1[0]);//10

arr1 将值的地址复制给arr2，修改arr2的值会影响到arr1的值。

克隆

JSON.stringify()

将[]或{}的对象转换成字符串形式的值

JSON.parse()

将字符串形式的[]或{}值转换成对象

克隆实现原理:

利用字符串赋值为传值的特点，先将对象转换成字符串形式，然后将字符串形式的值再转换成对象.
